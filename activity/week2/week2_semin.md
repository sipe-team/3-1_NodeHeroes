# 2주차 활동 정리
이번 주에는 제작할 패키지의 아이디어를 구체화하는 시간을 가졌습니다. 앞으로 제작할 패키지 개요는 다음과 같습니다.

## Convert Markdown to HTML 패키지
**패키지명**: 미정

### 선정 이유
이미 다양한 마크다운 라이브러리가 있지만, 좀 더 가볍고 심플한 패키지를 직접 제작해보고자 이 아이템을 선정했습니다.

향후 개인 블로그를 직접 제작할 계획이 있는데, 이 패키지를 활용해 쉽게 글을 작성하고 관리하고 싶습니다.

또한, 복잡한 파서를 작성하는 과정이 좋은 경험이 될 것이라고 생각했습니다.


### 주요 기능
1. **Markdown을 HTML로 변환**
   - `npx`로 파일 간 변환을 지원해 CLI 명령으로 쉽게 사용 가능하게 합니다.

2. **웹 컴포넌트 제작**
   - 마크다운을 `prop`으로 받아 HTML로 출력하는 웹 컴포넌트를 제공합니다.

3. **React 컴포넌트 제작**
   - 웹 컴포넌트를 활용해 React 환경에서도 쉽게 사용할 수 있는 컴포넌트를 제작합니다.

4. **기본 CSS 제공**
   - Bootstrap처럼 HTML 기본 스타일이 포함된 CSS를 제공하여 스타일링을 최소한으로 설정할 수 있습니다.

### 다른 라이브러리와의 차별점
`markdown-it`이나 `remark`와 달리, **심플한 기능과 사용 편의성**에 중점을 둔 패키지입니다.

기본 CSS 제공, 설정이 필요 없는 가벼운 사용 방식, 컴포넌트 제공을 통해 빠르고 직관적으로 사용할 수 있는 패키지로 차별화하고자 합니다.
